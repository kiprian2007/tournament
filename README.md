# Турнир вариантов

Одностраничное веб-приложение для ранжирования списков через попарное сравнение.  
Всё приложение — один файл: `tournament.html`. Нет зависимостей, нет сборки.

## Запуск

Открыть `tournament.html` в любом современном браузере. Сервер не нужен.

---

## Функциональность

- Создание именованных списков вариантов (до 30 пунктов)
- Редактирование и удаление списков
- Турнир: каждый вариант сравнивается с каждым (round-robin)
- Итоговый рейтинг от первого до последнего по числу побед
- Списки сохраняются в `localStorage` — переживают перезагрузку страницы

## Архитектура

### Экраны

Все экраны — `div.screen` внутри `<body>`. Видим только тот, у кого есть класс `.active`.  
Переключение через `showScreen(id)`.

| ID экрана | Назначение |
|---|---|
| `screen-home` | Список сохранённых наборов |
| `screen-edit` | Создание / редактирование набора |
| `screen-tournament` | Попарное сравнение |
| `screen-results` | Итоговый рейтинг |

### Модальное окно

`#overlay-delete` — подтверждение удаления. Открывается поверх текущего экрана.  
Закрывается кликом на фон или кнопкой «Отмена».

### Навигация

```
screen-home
  ├─ [+ Новый список]  → openEdit(null)  → screen-edit → saveList() → screen-home
  ├─ [Изменить]        → openEdit(id)    → screen-edit → saveList() → screen-home
  ├─ [Старт]           → startTournament(id)           → screen-tournament
  └─ [✕]              → openDeleteDialog(id)

screen-tournament
  └─ (все пары пройдены) → showResults()              → screen-results

screen-results
  ├─ [К спискам]       → goHome()                     → screen-home
  └─ [Ещё раз]         → playAgain() = startTournament(currentListId)
```

## Алгоритм турнира

### Round-robin (все пары)

Для `n` вариантов генерируются все уникальные пары `[i, j]` где `i < j`.  
Количество сравнений: `n × (n − 1) / 2`.

```js
for (let i = 0; i < items.length; i++)
  for (let j = i + 1; j < items.length; j++)
    pairs.push([i, j]);
```

### Перемешивание (Fisher-Yates)

Пары перемешиваются перед турниром, чтобы порядок сравнений был случайным:

```js
for (let i = pairs.length - 1; i > 0; i--) {
  const j = Math.floor(Math.random() * (i + 1));
  [pairs[i], pairs[j]] = [pairs[j], pairs[i]];
}
```

### Подсчёт результатов

Каждый вариант имеет счётчик побед `wins[index]`. После всех пар — сортировка по убыванию.  
При равном числе побед порядок сохраняется из оригинального списка.

### Защита от двойного клика

Флаг `choosing = true` устанавливается при выборе и сбрасывается через 300 мс  
(после анимации и перехода к следующей паре).

## Модель данных

### localStorage

**Ключ:** `tournament_lists`  
**Формат:** JSON-массив объектов списка

```json
[
  {
    "id": "lf3k2xab9r",
    "name": "Любимая еда",
    "items": "Пицца\nСуши\nБургер\nПаста\nРамен"
  }
]
```

| Поле | Тип | Описание |
|---|---|---|
| `id` | string | Уникальный ID: `Date.now().toString(36)` + random |
| `name` | string | Название списка, макс. 60 символов |
| `items` | string | Варианты через `\n`, уже дедуплицированы |

### Состояние турнира (только в памяти, не сохраняется)

```js
items        // string[] — варианты текущего турнира
pairs        // [number, number][] — перемешанные пары индексов
wins         // { [index]: number } — счётчики побед
pairIndex    // number — текущая пара
choosing     // boolean — дебаунс-флаг
currentListId // string — ID списка для «Ещё раз»
```

## Важные инварианты

> Изменение этих вещей сломает работающий код или сохранённые данные.

### DOM-идентификаторы (используются в JS)

`screen-home`, `screen-edit`, `screen-tournament`, `screen-results`,  
`lists-wrap`, `edit-title`, `edit-name`, `edit-items`, `edit-error`,  
`overlay-delete`, `delete-dialog-name`,  
`card-a`, `card-b`, `current-pair`, `total-pairs`, `progress-pct`, `progress-fill`,  
`result-title`, `result-list`, `btn-play-again`

### Ключ localStorage

`tournament_lists` — если переименовать, все сохранённые списки станут недоступны.

### Структура объекта списка

Поля `id`, `name`, `items` обязательны. `items` — строка с `\n` как разделителем.

### Алгоритм `choose(side)`

Принимает строго `'a'` или `'b'`. ID карточек должны оставаться `card-a` и `card-b`.

### Медали в результатах

Цвета первых трёх мест заданы через CSS `:nth-child(1/2/3)` — порядок в `result-list` важен.
